Glissez votre ROM vers le programme du système d'exploitation. Une ROM Smash Remix va alors être générée dans 
le dossier "Output".

Pour les utilisateurs de Project64 netplay (en ligne):
Si vous utilisez Project64KVE (icône Luma): remplacez Project64.rdb dans le dossier principal de l'émulateur par la version fournie dans cette "Patch" ou copier-coller ceci  dans votre Project64.rdb:

[05BA5CCD-ABB97C1E-C:45]
Good Name=SmashRemix{VERSION}
Internal Name=SmashRemix
Status=Compatible
Counter Factor=1
Culling=1
SMM-Cache=0
SMM-FUNC=0
SMM-TLB=0
Use TLB=No
RDRAM Size=8
ViRefresh=2200

Si vous utilisez Project64KVE (icône Kirby) Allez dans "File" > "Open Tools"... et "Check For Updates".
Si la mise à jour ne fonctionne pas, remplacez pj64.rdb dans le dossier Cfg par la version fournie dans cette "Patch"
-ou-
Téléchargez la dernière version de l'émulateur sur le site web: https://smash64.online:

Suivez les instructions générées par le programme si le générateur de "ROM" échoue.

Si ça ne fonctionne toujours pas, après avoir suivi les instructions, utilisez le "Delta UI". Suivez les instructions ci-dessous:

Étape 1: Vérifiez que vous avez la "ROM" Super Smash Bros 64. inchangée et légale. Vérifiez qu'elle se termine bien par "Z.64"
(.n64 ou .v64 ne fonctionneront pas!). Vous pouvez facilement convertir un de ces dossiers en un Z.64 par Tool64 (download here: https://www.zophar.net/utilities/n64aud/tool-n64.html). 
Après la conversion,  assurez-vous que la conversion a réussi en ouvrant le nouveau dossier dans Tool64

Étape 2: Démarrez le programme Delta UI. Dans la "Source File box" cliquez "open", ensuite sélectionnez votre fichier de "ROM" (Soyez certain qu'elle est en Z.64)

Étape 3: Dans la "Patch box" cliquez "Open", sélectionnez la version actuelle de smashremix1.5.0.xdelta (dans le dossier 'Patch')

Étape 4:  Dans le dossier "Output File Box" cliquez "Open", sélectionnez un nom pour le fichier qui se termine par .z64

Étape 5: Cliquez "Patch". Vous devriez avoir une copie fonctionnelle de Smash Remix

Pour les utilisateur d'android:

Pour les utilisateurs de MUPEN64PLUS:

Si vous voulez que la "ROM" apparaisse correctement sur l'interface d'utilisateur de MUPEN64PLUS (i.e. m64py) "Good ROM known database", copiez ce texte et collez-le sur votre
mupen64plus.ini actuel: 

[18EC45DD6C723AAB1F2D6577C68164F2]
GoodName=SmashRemix{VERSION}
CRC=05BAD793-42637A0F
Status=3
SaveType=SRAM
Players=4
Rumble=Yes
CountPerOp=1

Dans votre m64py pour Windows, ce fichier normalement dans le dossier "C:\Program Files (x86)\M64Py"
Si vous utilisez une autre interface d'utilisateur, vous pouvez essayer de chercher le fichier mupen64plus.ini dans le même dossier où l'interface est exécutable.
Dans Linux, ce fichier est normalement dans le dossier d'utilisateur "/home/your-username/.config/mupen64plus", par contre, ceci pourrait changer selon les distributions.
Vous n'êtes pas obligé de faire cette étape car mupen64plus fait déjà bien fonctionner Smash Remix, ça va seulement ajouter votre "ROM" dans les "Good ROM compatible database"(Base de données des bonne ROM compatible),
donc vous n'avez pas à chercher manuellement à toutes les fois.

La ligne "CountPerOp" de Mupen64Plus est l'équivalent de l'option "Counter Factor" de Project64.

De plus, ne pas ajouter la ligne "ViRefresh=2200" à mupen64plus.ini car mupen64plus utilise une différente méthode qui calcule automatiquement la fréquence d'actualisation du VI
pour le jeu qu'il fait fonctionner pour garantir la stabilité de la fréquence d'image en plus de garder une performance optimale, donc "l'overclocking" n'est pas nécessaire.
